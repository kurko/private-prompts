# 05 - Session Management

## Overview
Verify session lifecycle, token security, and state management resist hijacking,
fixation, and replay attacks.

## Tech Stack Adaptations

| Stack | Where to look |
|-------|---------------|
| Rails | `config/initializers/session_store.rb`, `config/application.rb` |
| Django | `settings.py` (SESSION_*), middleware config |
| Express/Node | `express-session` config, `cookie-session`, JWT middleware |
| Go | Session middleware config, cookie handling |

## Checklist Items

### SM-01: Session ID Entropy
- **What to check:** Session IDs have at least 128 bits of entropy
- **Detection patterns:**
  - Search: `session_store` configuration (Rails defaults are secure)
  - Search: `secret_key_base` is set and not a weak value
  - Search: `session.*secret|secret.*session` in config
  - Search: `uuid|SecureRandom|crypto\.randomBytes` in custom session generation
  - Files: `config/initializers/session_store.rb`, `config/secrets.yml`, `config/credentials.yml.enc`
- **Secure pattern:**
  ```ruby
  Rails.application.config.session_store :cookie_store, key: '_app_session'
  # secret_key_base should be at least 128 bits (auto-generated by Rails)
  ```
- **Severity:** High
- **CWE:** CWE-330
- **OWASP Top 10:** A07:2021-Identification and Authentication Failures

### SM-02: Secure Cookie Flags
- **What to check:** Session cookies have Secure, HttpOnly, SameSite attributes
- **Detection patterns:**
  - Search: `cookie.*httponly|httponly.*cookie` in session config
  - Search: `cookie.*secure|secure.*cookie` in session config
  - Search: `samesite` in session/cookie config
  - Search: `config\.force_ssl` (Rails sets Secure flag when true)
  - Files: `config/initializers/session_store.rb`, `config/environments/production.rb`
- **Secure pattern:**
  ```ruby
  Rails.application.config.session_store :cookie_store,
    key: '_app_session',
    secure: Rails.env.production?,
    httponly: true,
    same_site: :lax
  ```
- **Severity:** High
- **CWE:** CWE-614
- **OWASP Top 10:** A07:2021-Identification and Authentication Failures

### SM-03: Session Expiration
- **What to check:** Absolute and idle timeout configured
- **Detection patterns:**
  - Search: `expire_after` in session store config
  - Search: `session_length|session_timeout|timeout` in session config
  - Search: `Devise.timeout_in` (Devise timeoutable module)
  - Search: `maxAge|expires` in cookie/session config (Node)
  - Files: `config/initializers/session_store.rb`, `config/initializers/devise.rb`
- **Secure pattern:**
  ```ruby
  # Absolute timeout via session store
  Rails.application.config.session_store :cookie_store, expire_after: 12.hours
  # Idle timeout via Devise
  config.timeout_in = 30.minutes
  ```
- **Severity:** Medium
- **CWE:** CWE-613
- **OWASP Top 10:** A07:2021-Identification and Authentication Failures

### SM-04: Session Invalidation on Logout
- **What to check:** Server-side session destroyed on logout, not just cookie cleared
- **Detection patterns:**
  - Search: `reset_session` in logout/destroy action (Rails)
  - Search: `session\.destroy` or `req\.session\.destroy` (Node)
  - Search: `sign_out` or `logout` in controllers
  - Search: logout action that only clears cookies without server-side invalidation
  - Files: `app/controllers/sessions*.rb`, `routes/auth*.js`
- **Secure pattern:**
  ```ruby
  def destroy
    reset_session  # Destroys server-side session
    redirect_to root_path
  end
  ```
- **Severity:** High
- **CWE:** CWE-613
- **OWASP Top 10:** A07:2021-Identification and Authentication Failures

### SM-05: Session Invalidation on Password Change
- **What to check:** All sessions invalidated when password changes
- **Detection patterns:**
  - Search: `reset_session|destroy.*session` in password update flow
  - Search: `sign_out_all_scopes` (Devise)
  - Search: session version/generation that changes with password
  - Files: `app/controllers/**/*password*`, `app/models/user*.rb`
- **Secure pattern:**
  ```ruby
  def update_password
    @user.update(password_params)
    @user.update(session_token: SecureRandom.hex)  # Invalidates all sessions
    sign_in(@user, bypass: true)  # Re-sign in current session
  end
  ```
- **Severity:** High
- **CWE:** CWE-613
- **OWASP Top 10:** A07:2021-Identification and Authentication Failures

### SM-06: Concurrent Session Limits
- **What to check:** Maximum active sessions per user enforced
- **Detection patterns:**
  - Search: `concurrent_session|max_sessions|active_sessions` in models/config
  - Search: `devise-security` session_limitable in Gemfile
  - Search: session tracking table/model
  - Files: `app/models/user*.rb`, `Gemfile`, `db/migrate/**/*.rb`
- **Secure pattern:**
  ```ruby
  # Track sessions and limit concurrency
  class Session < ApplicationRecord
    belongs_to :user
    scope :active, -> { where("expires_at > ?", Time.current) }
  end
  ```
- **Severity:** Low
- **CWE:** CWE-613
- **OWASP Top 10:** A07:2021-Identification and Authentication Failures

### SM-07: Session Binding
- **What to check:** Sessions optionally bound to client fingerprint to detect hijacking
- **Detection patterns:**
  - Search: `user_agent|ip_address|fingerprint` stored with session
  - Search: session validation checking IP or user agent changes
  - Files: `app/controllers/application_controller.rb`, `app/models/session*.rb`
- **Secure pattern:**
  ```ruby
  # Store and validate session fingerprint
  before_action :validate_session_fingerprint
  def validate_session_fingerprint
    expected = Digest::SHA256.hexdigest("#{request.user_agent}#{request.remote_ip}")
    reset_session if session[:fingerprint] != expected
  end
  ```
- **Severity:** Low
- **CWE:** CWE-384
- **OWASP Top 10:** A07:2021-Identification and Authentication Failures

### SM-08: CSRF Token Per Session
- **What to check:** CSRF tokens tied to session, rotated on login
- **Detection patterns:**
  - Search: `protect_from_forgery` in ApplicationController (Rails - should be present)
  - Search: `csrf|_token|authenticity_token` in form handling
  - Search: `csurf` middleware (Node/Express)
  - Search: `CsrfViewMiddleware` (Django - should be enabled)
  - Files: `app/controllers/application_controller.rb`, `config/application.rb`
- **Secure pattern:**
  ```ruby
  class ApplicationController < ActionController::Base
    protect_from_forgery with: :exception
  end
  ```
- **Severity:** High
- **CWE:** CWE-352
- **OWASP Top 10:** A01:2021-Broken Access Control

### SM-09: Session Storage Security
- **What to check:** Server-side session data encrypted or stored securely
- **Detection patterns:**
  - Search: `session_store.*cookie` (check if encrypted - Rails 5.2+ encrypts by default)
  - Search: `session_store.*redis|session_store.*active_record` (server-side storage)
  - Search: `SESSION_ENGINE` (Django session backend)
  - Files: `config/initializers/session_store.rb`, `settings.py`
- **Secure pattern:**
  ```ruby
  # Rails 5.2+ encrypts cookie sessions by default
  # For sensitive data, use server-side storage
  Rails.application.config.session_store :redis_session_store,
    key: '_app_session', redis: { url: ENV['REDIS_URL'] }
  ```
- **Severity:** Medium
- **CWE:** CWE-311
- **OWASP Top 10:** A02:2021-Cryptographic Failures

### SM-10: Cookie Domain and Path Scoping
- **What to check:** Session cookies scoped to appropriate domain and path
- **Detection patterns:**
  - Search: `domain:` in session store config
  - Search: `path:` in session/cookie config
  - Search: cookie set with overly broad domain (e.g., `.example.com` when single subdomain)
  - Files: `config/initializers/session_store.rb`, `config/environments/**/*.rb`
- **Secure pattern:**
  ```ruby
  Rails.application.config.session_store :cookie_store,
    key: '_app_session',
    domain: 'app.example.com'  # Not .example.com
  ```
- **Severity:** Low
- **CWE:** CWE-1275
- **OWASP Top 10:** A05:2021-Security Misconfiguration

### SM-11: Session Data Encryption
- **What to check:** Sensitive data in session is encrypted, not just signed
- **Detection patterns:**
  - Search: `session\[.*password\]|session\[.*token\]|session\[.*secret\]` (sensitive data in session)
  - Search: `message_encryptor|MessageEncryptor` usage for session data
  - Search: `signed_cookie` vs `encrypted_cookie` usage
  - Files: `app/controllers/**/*.rb`, `config/initializers/session_store.rb`
- **Secure pattern:**
  ```ruby
  # Rails 5.2+ uses encrypted cookies by default
  # Don't store sensitive data in sessions at all if possible
  # If you must: session[:token] = encrypt(token)
  ```
- **Severity:** Medium
- **CWE:** CWE-311
- **OWASP Top 10:** A02:2021-Cryptographic Failures

### SM-12: Anti-Session-Riding (Double Submit Cookie)
- **What to check:** AJAX/API requests verify CSRF via header, not just cookie
- **Detection patterns:**
  - Search: `X-CSRF-Token|X-Requested-With` header validation
  - Search: `skip_before_action.*verify_authenticity_token` (Rails - danger)
  - Search: `csrf.*false|csrf.*disable` in API routes
  - Files: `app/controllers/**/*.rb`, `config/routes.rb`, `**/*api*`
- **Secure pattern:**
  ```ruby
  # Don't skip CSRF for browser-accessed APIs
  # Use token in header for AJAX
  # meta tag: csrf_meta_tags
  # JS: X-CSRF-Token header from meta tag
  ```
- **Severity:** High
- **CWE:** CWE-352
- **OWASP Top 10:** A01:2021-Broken Access Control

### SM-13: Token Refresh Mechanism
- **What to check:** Long-lived tokens use refresh rotation, not indefinite validity
- **Detection patterns:**
  - Search: `refresh_token|token_refresh|rotate` in auth/token code
  - Search: JWT with no expiration (`exp` claim missing)
  - Search: token validity > 24 hours without refresh mechanism
  - Files: `app/services/**/*.rb`, `lib/**/*.rb`, `**/*token*`, `**/*jwt*`
- **Secure pattern:**
  ```ruby
  # Short-lived access token (15 min) + refresh token (7 days)
  access_token = JWT.encode({ user_id: user.id, exp: 15.minutes.from_now.to_i }, secret)
  refresh_token = SecureRandom.hex(32)  # Stored server-side, rotated on use
  ```
- **Severity:** Medium
- **CWE:** CWE-613
- **OWASP Top 10:** A07:2021-Identification and Authentication Failures

### SM-14: Session Enumeration Prevention
- **What to check:** Session IDs cannot be enumerated or predicted
- **Detection patterns:**
  - Search: sequential or predictable session ID generation
  - Search: `auto_increment|sequence` in session table
  - Search: custom session ID generation not using SecureRandom
  - Files: `app/models/session*.rb`, `db/migrate/**/*session*`
- **Secure pattern:** Use framework defaults (Rails, Django, Express all use cryptographic randomness).
- **Severity:** High
- **CWE:** CWE-330
- **OWASP Top 10:** A07:2021-Identification and Authentication Failures

### SM-15: Cross-Site Session Sharing
- **What to check:** Sessions not shared across unrelated applications on same domain
- **Detection patterns:**
  - Search: `cookie.*domain.*\.` (wildcard domain cookie sharing)
  - Search: shared session store (Redis/DB) across different apps without namespace
  - Files: `config/initializers/session_store.rb`, `config/environments/**/*.rb`
- **Secure pattern:**
  ```ruby
  # Namespace session keys per application
  Rails.application.config.session_store :redis_session_store,
    key: '_myapp_session',  # Unique per app
    redis: { url: ENV['REDIS_URL'], key_prefix: 'myapp:session:' }
  ```
- **Severity:** Medium
- **CWE:** CWE-1275
- **OWASP Top 10:** A05:2021-Security Misconfiguration

### SM-16: Session Persistence Across Deploys
- **What to check:** Sessions survive application restarts (server-side store, not in-memory)
- **Detection patterns:**
  - Search: `MemoryStore|memory_store` in session config (bad for production)
  - Search: session store configuration for production environment
  - Files: `config/environments/production.rb`, `config/initializers/session_store.rb`
- **Secure pattern:**
  ```ruby
  # Use persistent store in production
  Rails.application.config.session_store :redis_session_store
  # Or :active_record_store, :cookie_store (stateless)
  ```
- **Severity:** Low
- **CWE:** CWE-613
- **OWASP Top 10:** A05:2021-Security Misconfiguration

### SM-17: WebSocket Session Binding
- **What to check:** WebSocket connections authenticated and bound to HTTP session
- **Detection patterns:**
  - Search: `ActionCable.*connection` authentication (Rails)
  - Search: `ws.*auth|auth.*ws|socket.*auth` in WebSocket handlers
  - Search: `verified_user|find_verified_user` in cable connection
  - Files: `app/channels/application_cable/connection.rb`, `**/*socket*`, `**/*cable*`
- **Secure pattern:**
  ```ruby
  module ApplicationCable
    class Connection < ActionCable::Connection::Base
      identified_by :current_user
      def connect
        self.current_user = find_verified_user || reject_unauthorized_connection
      end
    end
  end
  ```
- **Severity:** High
- **CWE:** CWE-287
- **OWASP Top 10:** A07:2021-Identification and Authentication Failures

### SM-18: Mobile Session Management
- **What to check:** Mobile API tokens stored securely (keychain/keystore, not shared prefs)
- **Detection patterns:**
  - Search: `SharedPreferences.*token|UserDefaults.*token` (insecure mobile storage)
  - Search: `Keychain|KeyStore|SecureStorage` (secure mobile storage)
  - Search: mobile token endpoints without device binding
  - Files: `**/*.swift`, `**/*.kt`, `**/*.java`, `**/*mobile*`
- **Secure pattern:** Store tokens in platform keychain/keystore, not in plain storage.
- **Severity:** High
- **CWE:** CWE-312
- **OWASP Top 10:** A07:2021-Identification and Authentication Failures
